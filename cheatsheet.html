Nginx

Documentation: https://nginx.org/en/docs/

Remove .html extensions from urls:

    location / {
        if ($request_uri ~ ^/(.*)\.html(\?|$)) {
            return 302 /$1;
        }
        try_files $uri $uri.html $uri/ =404;


Redirect requests for other domains to your domain:

    if ($host != example.com) {
        return 301 https://example.com$request_uri;
    }


Redirect http to https:

    if ($host = example.com) {
        return 301 https://$host$request_uri;
    }

SSH

Documentation: https://www.openssh.com/manual.html

Generate SSH key:

ssh-keygen


Add pubkey to authorized_keys file on remote machine:

ssh-copy-id root@yourdomain.com


Typical file permissions:

chmod 700 ~/.ssh/
chmod 644 ~/.ssh/id_rsa.pub
chmod 600 ~/.ssh/id_rsa
chmod 644 ~/.ssh/authorized_keys


Minimum sshd_config options to disable Password Authentication:

PasswordAuthentication no
ChallengeResponseAuthentication no
UsePAM no


Config options for multiple hosts (Where "Host *" represents all other connecitons.):

~/.ssh/config
/etc/ssh/ssh_config


Host server1
  HostName hostname.example.com
  User admin
  IdentityFile ~/.ssh/id_rsa

Host server2
  HostName example2.com
  Port 2222
  User john
  IdentityFile ~/.ssh/id_rsa2

Host server3
  HostName 192.168.1.100
  User root
  IdentityFile ~/.ssh/id_rsa3

Host *
  IdentityFile /path/to/some/other/key

ASD recommended security configurations (ISM-0484):

only listen on the required interfaces (ListenAddress xxx.xxx.xxx.xxx)
have a suitable login banner (Banner x)
have a login authentication timeout of no more than 60 seconds (LoginGraceTime 60)
disable host-based authentication (HostbasedAuthentication no)
disable rhosts-based authentication (IgnoreRhosts yes)
disable the ability to login directly as root (PermitRootLogin no)
disable empty passwords (PermitEmptyPasswords no)
disable connection forwarding (AllowTCPForwarding no)
disable gateway ports (GatewayPorts no)
disable X11 forwarding (X11Forwarding no)

Tools

Rsync

Documentation: https://rsync.samba.org/documentation.html

# uploading
rsync -rtvzP /path/to/file root@example.org:/path/on/the/server

# downloading
rsync -rtvzP root@example.org:/path/to/file /path/to/file

-r: run recurssively (include directories)
-t: transfer modification times, which allows skipping files that have not been modified on future uploads
-v: visual, show files uploaded
-z: compress files for upload
-P: if uploading a large file and upload breaks, pick up where we left off rather than reuploading the entire file

Tar

Documentation: https://www.gnu.org/software/tar/manual/

# create archive
tar -czvf name-of-archive.tar.gz /path/to/directory-or-file /optional/file-or-directory

-c: Create an archive.
-z: Compress the archive with gzip.
-v: Display progress in the terminal while creating the archive, also known as "verbose" mode. The v is always optional in these commands, but it's helpful.
-f: Allows you to specify the filename of the archive.

--exclude to exclude specific files or directories.


# extract archive
tar -xzvf archive.tar.gz

# into a specific directory
tar -xzvf archive.tar.gz -C /tmp
